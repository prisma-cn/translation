---
title: 启用本地数据库函数的扩展功能
metaDecription: 如何为使用Prisma Migrate的项目启用本地数据库函数扩展功能。
---

<TopBlock>

<Admonition type="warning">

**不支持 MongoDB** <br />
Prisma Migrate 目前不支持 [MongoDB connector](../../../concepts/database-connectors/mongodb)。

</Admonition>

有一些[本地数据库函数](/concepts/components/prisma-schema/features-without-psl-equivalent#native-database-functions) <span class="concept"></span> are part of optional extensions. You cannot use the function if the extension is not installed. If your project uses [Prisma Migrate](../../../concepts/components/prisma-migrate), 必须安装该扩展作为迁移的一部分。

<Admonition type="warning">

**不要** 手动安装扩展 - [影子数据库](../../../concepts/components/prisma-migrate/shadow-database)需要同样的扩展，由于这个数据库是自动创建和删除的，所以安装扩展的唯一方法是使用迁移。

</Admonition>

下面的例子演示了如何安装`pgcrypto`扩展作为迁移的一部分：

1. 将具有本地数据库功能的字段添加到 schema 中：

   ```prisma
   model User {
     id String @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
   }
   ```

   如果包含操作符(例如`::TEXT`)，必须把整个函数用括号包起来:

   ```prisma
   @default(dbgenerated("(gen_random_uuid()::TEXT)")
   ```

1. 使用`--create-only`标志来生成一个新的未执行的迁移:

   ```terminal
   npx prisma migrate dev --create-only
   ```

1. 打开生成的`migration.sql`文件，启用`pgcrypto`模块:

   ```sql
   CREATE EXTENSION IF NOT EXISTS pgcrypto;

   ADD COLUMN     "id" UUID NOT NULL DEFAULT gen_random_uuid(),
   ADD PRIMARY KEY ("id");
   ```

1. 应用迁移:

   ```terminal
   npx prisma migrate dev
   ```

每次你重置数据库或向你的团队添加新成员时，所有需要的函数都是迁移历史的一部分。

</TopBlock>
